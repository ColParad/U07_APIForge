# Contributing to APIForge Core

Thank you for your interest in APIForge Core.

## Current repository phase

This is a clean public repository being prepared for the reviewed open-source Core distribution. Runtime source submissions should not be opened until the initial Core export and build boundary have been published.

## Public contribution boundary

Before submitting a change, read [`docs/public/OPEN_CORE_SCOPE.md`](docs/public/OPEN_CORE_SCOPE.md).

Contributions must not contain:

- credentials, tokens, private endpoints, environment values, or deployment details;
- internal architecture, licensing, billing, subscription, authentication, entitlement, or security-audit documents;
- source code copied from private or separately distributed modules;
- unreleased product plans or internal execution materials.

## Security reports

Do not open a public issue for suspected vulnerabilities or accidental disclosure of internal information. Follow [`SECURITY.md`](SECURITY.md).

## Future code contributions

Code contribution and build instructions will be added once the independently buildable Core source distribution has been published and validated.
