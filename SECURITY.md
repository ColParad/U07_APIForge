# Security Policy

## Reporting a vulnerability

Please do **not** open a public issue for a suspected security vulnerability, credential exposure, authentication issue, or other disclosure-sensitive report.

Use GitHub's private security reporting feature for this repository when it is enabled, or contact the project owner through the private contact channel published on the PolyCode website.

## Public repository scope

This repository is intended to contain only reviewed public APIForge Core material. It must not contain internal platform architecture, deployment details, private service contracts, credentials, tokens, security audits, or commercial implementation records.

If you observe material in this public repository that appears to expose credentials or internal operational details, report it privately rather than discussing it in a public issue.

## Supported versions

The public Core source distribution is not yet released. Supported-version reporting will be added with the first published Core release.
